# Curso 25/26. Práctica 3. Interfaces gráficas para aplicaciones web

## Welcome :wave:

- **Who is this for**: Grupos de prácticas de la asignatura _IPM_.

- **What you'll learn**: Implementación de interfaces gráficas usando
  lenguajes propios del desarrollo Web. Diseño mobile-first,
  adaptativo y accesible.

- **What you'll build**: Construiréis una interfaz para aplicaciones
  web.

- **Prerequisites**: Asumimos que os resultan familiares el
  funcionamiento de una aplicación web y las herramientas básicas para
  desarrolladores web.

- **How long**: Este assigment está formado por tres pasos o
  _tareas_. La duración estimada de cada tarea es de una semana
  lectiva.


La práctica continúa tomando como referencia la aplicación de
compartición de gastos para la que has desarrollado varias versiones
en prácticas anteriores. Sin embargo el objetivo de esta práctica no
es el desarrollo de la aplicación completa. El objetivo es apredender
y aplicar las tecnologías propias del desarrollo de interfaces de
usuaria en aplicaciones web, a saber:

  - Uso de los lenguajes html5, css3 y javascript.

    Puesto que nos enfocamos en el uso de estos lenguajes, no se
    empleará ninguna librería ni framework.
	
  - Creación de documentos estándares.
  
  - Creación de html semántico.
  
  - Desarrollo de interfaces accesibles.
  
  - Desarrollo _mobile first_ y adaptativo.

  - Uso de las herramientas que nos permitan comprobar el desempeño
    adecuado en los puntos anteriores.
	

<details id=1>
<summary><h2>Tarea 1: Implementar una interfaz</h2></summary>

En este ejercicio tienes que implementar una interfaz únicamente para
la parte descrita a continuación. No debes implementar ningún tipo de
funcionalidad de la aplicación.

La parte a implementar es un informe en que se muestran todos los
datos correspondientes a los gastos compartidos (viaje, cena,
cumpleaños, ...). Los datos mostrados en el informe se pueden obtener
por cualquier medio que consideres oportuno.


### :wrench: Esta tarea tiene las siguientes partes:

  1. Diseña la interfaz de la aplicación siguiendo estas pautas:

     - El diseño tiene que seguir una metodología _mobile first_.
	 
	 - El diseño tiene que se adaptativo. Considera que la usuaria
       puede estar usando un navegador de teléfono móvil, tablet o
       escritorio.

     - El diseño tiene que estar contenido en un único documento o
       página web.

  2. Añade al repositorio un fichero _PDF_ con el nombre
     `diseño-iu.pdf` donde se muestre el diseño que vas a implementar.

  3. Implementa el diseño empleando los lenguajes _html5_ y _css3_
     siguiendo las pautas del W3C:
	 
       - Usa html semántico.
	 
       - Separa el contenido (html5) de la presentación (css3).
	 
	   - La implementación debe seguir las normas de accesibilidad [WCAG
         2](https://www.w3.org/WAI/standards-guidelines/wcag/).
		 
	   - Todos los elementos de html5 y css3 empleados tienen que
         estar en el
         [Baseline](https://web-platform-dx.github.io/web-features/)

     Recuerda que no debes usar ninguna librería o framework CSS.
	   

### :books: Objetivos de aprendizaje:

  - Diseño _mobile-first_.
  
  - Diseño adaptativo.
  
  - Uso de estándares web.
  
  - HTML semántico.
  
  - Accesibilidad web.
  
</details>


<details id=2>
<summary><h2>Tarea 2: Interactividad</h2></summary>

A lo largo de esta tarea debes respetar las condiciones establecidas
en las tareas anteriores.

En esta tarea tienes que complementar la tarea anterior añadiendo la
posibilidad de modificar los datos de los gastos mostrados.

De forma similar a lo establecido en el punto anterior, no es
necesario enviar los datos a ningún servidor concreto. Incluso es
posible simular su envío.

### :wrench: Esta tarea tiene las siguientes partes:

  1. Extiende el diseño de la interfaz para incluir los elementos
     necesarios para implementar las funcionalidades requeridas.
	 
  2. Modifica la implementación de la tarea anterior para añadir las
     funcionalidades requeridas.
	 
	 
### :books: Objetivos de 
  - javascript en el navegador.
  
  - DOM.
  
</details>



<details id=3>
<summary><h2>Tarea 3: WAI-ARIA</h2></summary>

A lo largo de esta tarea debes respetar las condiciones establecidas
en las tareas anteriores.


En esta tarea tienes que asegurarte de que tu implementación se 
ajuste a las normas de accesibilidad en aplicaciones Web
[WAI-ARIA](https://www.w3.org/WAI/standards-guidelines/aria/).
	 
Dado que en la tarea 1 ya te ocupaste de la accessiblidad, en esta
tarea el mayor esfuerzo se centará en los resultados de la tarea 2.
Esto no quiere decir que no sea necesario reevaluar los cambios
introducidos sobre la tarea 1.


### :wrench: Esta tarea tiene las siguientes partes:

  1. Modifica la implementación para corregir aquellas partes que no
     se ajusten a las normas de accesibilidad.
	 
  2. Crea un informe con un listado de pruebas exploratorias dirigidas
     a validar la accessiblidad de la interfaz web que has creado.
	 
  3. Realiza las pruebas que has descrito y corrige los fallos
     detectados.
	 
### :books: Objetivos de aprendizaje:

  - Accesibilidad en aplicaciones web.
